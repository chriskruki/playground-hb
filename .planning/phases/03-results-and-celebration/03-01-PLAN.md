<task type="auto">
  <name>Task 1: Create ResultsScreen component with rankings, over/under, and winner celebration</name>
  <files>score/src/components/results-screen.tsx</files>
  <action>
Create `score/src/components/results-screen.tsx` as a "use client" component following the same pattern as ScoringScreen (receives `holesData: HoleData[]` prop, reads state from Zustand). Use `import type { HoleData } from "@/lib/holes"` for the server/client boundary.

**Ranking computation:**
- Compute `coursePar` by summing all `holesData[i].par`
- For each player, compute `totalStrokes` by summing `scores[player.id][i]` for all 9 holes, falling back to `holesData[i].par` for any missing/undefined entries (defensive against NaN)
- Compute `overUnder = totalStrokes - coursePar`
- Sort ascending by totalStrokes (lowest wins)
- Assign rank numbers: rank 1 for first, but if two adjacent players have the same totalStrokes, they share the same rank (tie handling)
- Determine winners: all players with rank === 1 are winners (handles ties)

**Layout (mobile-first, max-w-md like rest of app):**
- Header: "Results" title centered, course par total displayed below (e.g., "Course Par: 27")
- Player ranking list: Each player in a Card-like row showing: rank number (1st/2nd/3rd with emoji or Trophy icon from lucide-react for rank 1), player name, total strokes, over/under display using the established color pattern (red for over, green for under, muted "E" for even)
- Winner row(s): Distinguished with accent/gold border (`border-accent` or `border-primary`), Trophy icon from lucide-react, and a Framer Motion spring scale animation
- "Play Again" button at bottom: full-width Button calling `goToLanding()` from the store

**Framer Motion animations:**
- Container: `motion.div` with `variants` using `staggerChildren: 0.15` for the player list
- Each player row: `motion.div` with variants `hidden: { opacity: 0, y: 20 }` / `visible: { opacity: 1, y: 0 }`
- Winner row(s): Enhanced variant with `scale` spring animation: `hidden: { opacity: 0, scale: 0.8, y: 20 }` / `visible: { opacity: 1, scale: 1, y: 0, transition: { type: "spring", bounce: 0.5 } }`
- Optional: 8-12 small confetti-like `motion.div` elements above the winner row with randomized keyframes (small colored circles/squares that animate outward and fade). Keep it lightweight -- just CSS-sized divs with random positions/rotations, not a canvas system.

**Imports needed:** framer-motion (motion), lucide-react (Trophy), useGameStore from "@/lib/game-store", Button and Card/CardContent from ui components, type HoleData from "@/lib/holes".

Do NOT use router.push or any URL navigation -- this app uses a single-URL state machine. Do NOT import holes.ts at runtime (only the type).
