  <action>
Modify `score/src/components/app-shell.tsx` to apply par overrides from localStorage before passing holesData to child screens.

1. Import `applyParOverrides` from `@/lib/par-overrides`.
2. Add state: `const [effectiveHoles, setEffectiveHoles] = useState(holesData);`
3. Add a `useEffect` that runs after hydration to merge overrides:
```typescript
useEffect(() => {
  setEffectiveHoles(applyParOverrides(holesData));
}, [holesData]);
```
This runs client-side only, avoiding hydration mismatch (server renders with markdown pars, client updates with localStorage overrides after mount — same pattern as the existing hydration guard).

4. Pass `effectiveHoles` instead of `holesData` to `ScoringScreen` and `ResultsScreen`:
```typescript

```

Do NOT modify any other components. The par override is transparent — ScoringScreen and ResultsScreen already consume `holesData` prop and use `hole.par` from it. Changing the prop at the source is sufficient.

Do NOT put admin state in the Zustand game store (per research anti-pattern). Admin config is separate from game state.
